# Portfolio Strategy

## Goal

Maximize Build Small Hackathon award chances by submitting a portfolio of
finished projects rather than a single broad app.

## Constraints

- Each model must be under 32B total parameters.
- Submissions must be Gradio Spaces under the Build Small Hackathon org.
- Each submission needs a demo video and social post proof.
- Each project should map to at least one main track, one sponsor surface, and
  one bonus or special award surface.

## Award targeting

| Award surface | Strategy |
| --- | --- |
| Backyard AI | Practical apps for document help, tutoring, and developer workflows. |
| Thousand Token Wood | Whimsical creative apps with strong demo value. |
| Sponsor prizes | Build each app around at least one sponsor model or platform. |
| Bonus Quest Champion | Stack model size, local/small model, fine-tuning, custom UI, and agent badges where practical. |
| Best Demo | Prioritize simple, visual before/after stories. |
| Best Agent | Use explicit multi-step workflows where the agent plan is visible. |
| Community Choice | Create shareable outputs and clear social posts. |

## Modal credit use

Modal credits are best used for:

- Fine-tuning or LoRA experiments for visual/style projects.
- Batch parsing or model experiments for document intelligence.
- Sandboxed code execution for `Roast My Repo`.
- Temporary GPU inference where HF ZeroGPU is not enough.

Avoid burning Modal credits on ordinary app hosting unless it materially improves
a sponsor award or demo.
